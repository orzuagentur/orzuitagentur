# Coding Rules

## General Rules

* use TypeScript only
* use App Router
* use reusable components
* use modular architecture
* avoid duplicated code
* use scalable patterns

---

## Styling Rules

* use Tailwind CSS
* no inline styles
* use consistent spacing
* use responsive design
* mobile-first approach

---

## Animation Rules

* use Framer Motion
* smooth transitions only
* premium animation timing
* avoid excessive motion
* support reduced motion accessibility

---

## Component Rules

* split large components
* use semantic HTML
* accessible UI
* clean props structure
* reusable sections

---

## Backend Rules

* validate all forms
* secure server actions
* separate business logic
* use environment variables
* sanitize user input

---

## Database Rules

* use proper table relations
* use timestamps
* use typed queries
* scalable schema design

---

## Admin Dashboard Rules

* modular dashboard pages
* reusable cards
* analytics widgets
* dynamic CMS editing
* protected routes

---

## SEO Rules

* proper metadata
* semantic headings
* optimized performance
* OpenGraph support
* sitemap support

---

## Performance Rules

* optimize images
* lazy load heavy sections
* dynamic imports for 3D
* avoid unnecessary rerenders

---

## Code Quality

* production-ready code
* readable naming

* clean commits
